- **Author**: Graduate student (MSCSDA program)
- **Framework**: IBM Qiskit (v2.1+) with IBM Quantum Runtime
- **Hardware**: IBM Marrakesh (156-qubit processor), with auto-fallback to least-busy backend
- **IEEE Paper**: https://www.overleaf.com/read/mnhqjpfqdzgs#8afe03

## Key Functions — Shor (Qiskit.ipynb)

- **`build_shor_circuit(num_control, num_target=4, a=2, N=15)`** → QuantumCircuit
- **`transpile_for_backend(circuit, backend, optimization_level, seed)`** → (transpiled_circuit, metrics_dict)
- **`run_ideal(circuit, shots=100_000, seed)`** → {bitstring: probability}
- **`run_noisy(circuit, backend, shots, seed)`** → {bitstring: probability} (uses AerSimulator with device noise model or generic depolarizing fallback)
- **`run_hardware(transpiled_circuit, backend, shots, resilience_level, dd_enable)`** → (distribution_dict, job_id)
- **`compute_metrics(dist, ideal_dist, noisy_dist, num_control, ...)`** → {tvd_vs_ideal, tvd_vs_noisy, success_prob, ...}

## Key Functions — Grover (Grover.ipynb)

- **`build_oracle(num_qubits, marked_state)`** → Gate (phase-flip oracle via X-flips + multi-controlled Z)
- **`build_diffusion(num_qubits)`** → Gate (Grover diffusion: H-X-MCZ-X-H)
- **`optimal_num_iterations(num_qubits, num_marked=1)`** → int (floor(π/4 × √(N/M)))
- **`theoretical_success_prob(num_qubits, num_iterations)`** → float (sin²((2k+1)θ))
- **`build_grover_circuit(num_qubits, marked_state)`** → QuantumCircuit
- **`compute_grover_metrics(dist, ideal_dist, noisy_dist, num_qubits, marked_state)`** → {tvd_vs_ideal, success_prob, amplification, ...}
- Reuses verbatim: `transpile_for_backend`, `run_ideal`, `run_noisy`, `run_hardware`

## Experiment Design

**Three benchmarking tiers** (identical for both algorithms):
- **Tier A (Ideal)**: StatevectorSampler, 100k shots, noiseless baseline
- **Tier B (Noisy)**: AerSimulator with backend-derived noise model (or generic ~0.1%/1% depolarizing)
- **Tier C (Hardware)**: IBM SamplerV2 on real quantum processor

**Four sweep axes** (48 total configs each):
- Transpilation optimization level: [0, 1, 2, 3]
- **Shor**: Control-qubit precision [4, 6, 8] → total qubits [8, 10, 12]
- **Grover**: Search-register qubits [2, 3, 4] → search space [4, 8, 16], iterations [1, 2, 3]
- Resilience level: [0, 1]
- Dynamical decoupling: [Off, On] (XpXm sequence)

**Metrics**:
- **TVD** (Total Variation Distance): 0=identical, 1=disjoint
- **Success probability (Shor)**: probability mass within ε-window of expected peaks {0, 0.25, 0.5, 0.75}
- **Success probability (Grover)**: probability of measuring the single marked state
- **Amplification (Grover only)**: success_prob / (1/2^n) — fold-improvement over random guessing

## MCP Servers (configured in .mcp.json)

Six Qiskit MCP servers are configured, all using binaries from `.venv/bin/`:
1. **qiskit-mcp** — circuit analysis, transpilation, QASM conversion
2. **qiskit-docs** — documentation search, guides, SDK module docs
3. **qiskit-ibm-runtime** — backend listing, job submission, calibration data
4. **qiskit-ibm-transpiler** — AI-powered routing, synthesis, transpilation
5. **qiskit-code-assistant** — Qiskit code generation and RAG Q&A
6. **qiskit-gym** — RL-based circuit synthesis environments

Servers 3, 4, 5 use `QISKIT_IBM_TOKEN` from apikey.json.
